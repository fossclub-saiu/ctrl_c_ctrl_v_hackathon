# General Rules & Guidelines

## Participation Rules

### 1. One Problem Statement Per Team ✅
Each team must submit work on only **ONE** problem statement. Teams found submitting work on multiple problems will be disqualified.

### 2. Level Selection ✅

| Your Year | Eligible Levels |
|-----------|-----------------|
| First Year | Level 1, 2, or 3 |
| Second Year | Level 2 or 3 |
| Third Year | Level 3 only |

!!! tip "Choose Wisely"
    Pick a level based on your comfort and learning goals. First years can challenge themselves with higher levels!

### 3. Team Composition ✅
- Teams can have **1-4 members**
- Solo participation is allowed and encouraged

### 4. Original Work ✅
- Solutions must demonstrate **original logic and implementation**
- Using frameworks and APIs is allowed, but core business logic must be your own
- Code copied directly from tutorials without understanding will be penalized

!!! warning "Important"
    During evaluation, you may be asked to explain any part of your code. Make sure you understand what you've written!

### 5. Technology Stack ✅
- **No restrictions** on programming languages, frameworks, or tools
- Use whatever you're comfortable with or want to learn

### 6. AI Tools Usage ✅

| Allowed | Notes |
|---------|-------|
| AI APIs (ChatGPT, Claude, Gemini, etc.) as wrappers | ✅ |
| Pre-trained models and cloud APIs | ✅ |
| Code suggestions from AI assistants | ✅ (with understanding) |
| Training ML models from scratch | Not required |

!!! info "AI Usage Policy"
    Using AI for code suggestions is allowed, but you must understand every line of code you submit. Blind copy-paste without understanding will be penalized during evaluation.
